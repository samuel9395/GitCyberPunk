Introdução ao Git e GitHub

Git é um sistema de controle de versão distribuído que permite rastrear mudanças no código fonte durante o desenvolvimento de software. GitHub é uma plataforma de hospedagem de código que utiliza o Git.

DICA PRO: Este guia assume que você já tem o Git instalado em seu computador. Se não tiver, baixe em git-scm.com.

Por que usar Git?

O Git permite que você:

  • Controle o histórico de versões do seu código
  • Trabalhe em equipe de forma eficiente
  • Experimente novas ideias sem risco
  • Recupere versões anteriores facilmente

Configuração Inicial

1. Configurar usuário e email

git config --global user.name "Seu Nome"
Define o nome de usuário globalmente para todos os repositórios.
git config --global user.email "seu@email.com"
Define o email do usuário globalmente.

2. Verificar configurações

git config --list
Lista todas as configurações do Git.

Criando e Clonando Repositórios

1. Inicializar um repositório local

git init
Cria um novo repositório Git no diretório atual.

2. Clonar um repositório existente

git clone https://github.com/usuario/repositorio.git
Baixa um repositório remoto para o seu computador.
git clone https://github.com/usuario/repositorio.git nome-da-pasta
Clona o repositório em uma pasta com nome específico.

Trabalhando com Alterações

1. Verificar status do repositório

git status
Mostra o estado atual do diretório de trabalho e da área de staging.

2. Adicionar arquivos para commit

git add nome-do-arquivo
Adiciona um arquivo específico à área de staging.
git add .
Adiciona todos os arquivos modificados e novos à área de staging.
git add -A
Adiciona todos os arquivos (modificados, novos e deletados) à área de staging.

3. Fazer commit das alterações

git commit -m "Mensagem descritiva do commit"
Registra as alterações no repositório com uma mensagem.
git commit -a -m "Mensagem descritiva"
Adiciona automaticamente todos os arquivos rastreados e faz commit.

4. Ver histórico de commits

git log
Mostra o histórico de commits.
git log --oneline
Mostra o histórico de commits de forma resumida.
git log --graph --oneline
Mostra o histórico com gráfico de branches.

Trabalhando com Branches

1. Listar branches

git branch
Lista todas as branches locais.
git branch -a
Lista todas as branches (locais e remotas).

2. Criar uma nova branch

git branch nome-da-branch
Cria uma nova branch.

3. Trocar de branch

git checkout nome-da-branch
Muda para a branch especificada.
git checkout -b nome-da-branch
Cria uma nova branch e muda para ela.

4. Mesclar branches

git merge nome-da-branch
Mescla a branch especificada na branch atual.

5. Deletar branch

git branch -d nome-da-branch
Deleta a branch local especificada.

Trabalhando com Repositórios Remotos

1. Adicionar repositório remoto

git remote add origin https://github.com/usuario/repositorio.git
Adiciona um repositório remoto com o nome "origin".

2. Ver repositórios remotos

git remote -v
Lista todos os repositórios remotos configurados.

3. Baixar alterações do repositório remoto

git fetch origin
Baixa as alterações do repositório remoto, mas não as mescla.
git pull origin main
Baixa as alterações da branch main e as mescla na branch atual.

4. Enviar alterações para o repositório remoto

git push origin main
Envia os commits locais da branch main para o repositório remoto.
git push -u origin nome-da-branch
Envia uma branch local para o repositório remoto e configura o tracking.

Desfazendo Alterações

1. Desfazer alterações em arquivos não commitados

git checkout -- nome-do-arquivo
Descarta alterações locais em um arquivo específico.
git restore nome-do-arquivo
Versão mais recente do comando para descartar alterações.

2. Remover arquivos da área de staging

git reset HEAD nome-do-arquivo
Remove um arquivo da área de staging, mantendo as alterações no diretório de trabalho.
git restore --staged nome-do-arquivo
Versão mais recente do comando para remover da staging.

3. Alterar o último commit

git commit --amend -m "Nova mensagem"
Altera a mensagem do último commit.

4. Reverter um commit

git revert hash-do-commit
Cria um novo commit que desfaz as alterações de um commit específico.

5. Resetar para um commit anterior

git reset --soft hash-do-commit
Move HEAD para um commit anterior, mantendo as alterações na área de staging.
git reset --hard hash-do-commit
Move HEAD para um commit anterior e descarta todas as alterações posteriores.
Atenção: Use git reset --hard com cuidado, pois ele descarta permanentemente as alterações não commitadas.

Fluxo de Trabalho Típico

Fluxo Básico de Desenvolvimento

1
Atualizar seu repositório local
git pull origin main
2
Criar uma nova branch para sua funcionalidade
git checkout -b minha-nova-funcionalidade
3
Fazer suas alterações e commits
git add .
git commit -m "Implementa nova funcionalidade"
4
Enviar a branch para o repositório remoto
git push -u origin minha-nova-funcionalidade
5
Criar um Pull Request no GitHub

Vá para o repositório no GitHub e crie um Pull Request para mesclar sua branch com a main.

6
Após aprovação, mesclar e atualizar local
git checkout main
git pull origin main
git branch -d minha-nova-funcionalidade

Comandos Avançados

1. Stash (guardar alterações temporariamente)

git stash
Guarda as alterações não commitadas temporariamente.
git stash list
Lista todos os stashes.
git stash apply
Aplica o stash mais recente.
git stash pop
Aplica o stash mais recente e o remove da lista.

2. Tags

git tag nome-da-tag
Cria uma tag no commit atual.
git tag -a v1.0 -m "Versão 1.0"
Cria uma tag anotada.
git push origin --tags
Envia todas as tags para o repositório remoto.

3. Diferenças entre commits

git diff
Mostra as diferenças entre o diretório de trabalho e a área de staging.
git diff --staged
Mostra as diferenças entre a área de staging e o último commit.
git diff hash1 hash2
Mostra as diferenças entre dois commits específicos.

Resolução de Conflitos

1. Identificar conflitos

Quando ocorre um conflito durante um merge ou rebase, o Git marcará os arquivos com conflitos.

2. Resolver conflitos manualmente

Abra os arquivos com conflitos e procure por marcadores como:

<<<<<<< HEAD
Seu código local
=======
Código do repositório remoto
>>>>>>> branch-remota

Edite o arquivo para resolver o conflito, removendo os marcadores e mantendo o código desejado.

3. Finalizar a resolução

git add nome-do-arquivo
Adiciona o arquivo com conflito resolvido.
git commit
Finaliza o merge após resolver todos os conflitos.
